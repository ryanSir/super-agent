## ADDED Requirements

### Requirement: 亮色/暗色主题切换按钮
Header 区域 SHALL 提供一个主题切换按钮，点击后在亮色（白天）和暗色（夜晚）主题之间切换。

#### Scenario: 默认暗色主题
- **WHEN** 用户首次打开应用（无 localStorage 记录）
- **THEN** 应用以暗色主题显示

#### Scenario: 切换到亮色主题
- **WHEN** 用户点击主题切换按钮（当前为暗色）
- **THEN** 界面立即切换为亮色主题，按钮图标变为月亮（表示可切回暗色）

#### Scenario: 切换到暗色主题
- **WHEN** 用户点击主题切换按钮（当前为亮色）
- **THEN** 界面立即切换为暗色主题，按钮图标变为太阳（表示可切回亮色）

### Requirement: 主题偏好持久化
用户的主题选择 SHALL 持久化到 localStorage，下次打开应用时自动恢复。

#### Scenario: 主题偏好恢复
- **WHEN** 用户刷新页面或重新打开应用
- **THEN** 应用自动应用上次选择的主题，无需重新切换

### Requirement: 亮色主题 CSS 变量
亮色主题 SHALL 覆盖所有颜色 CSS 变量，提供清晰的白底深字配色方案。

#### Scenario: 亮色主题背景色
- **WHEN** 亮色主题激活
- **THEN** 页面背景为白色/浅灰，文字为深色，气泡和卡片使用浅色背景

#### Scenario: 暗色主题背景色
- **WHEN** 暗色主题激活
- **THEN** 页面背景为深蓝黑色，文字为浅色（保持原有配色不变）
